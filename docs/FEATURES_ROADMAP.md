# Lokal Pikol — Features Roadmap

## Directory UX Improvements

| # | Feature | Status |
|---|---------|--------|
| 1 | **Text search by court name** — Allow users to search for a specific court by name. Currently there is no way to find a known court without scrolling through the entire directory. | To-Do |
| 2 | **Listing count indicator** — Display "Showing X of Y courts" above the grid that updates as filters change. Gives users immediate context about directory size and whether their filters are too narrow. | To-Do |
| 3 | **"New" badge on recent listings** — A small visual badge on listing cards added within the last 14 days. Creates a sense of freshness, rewards new court registrations with visibility, and gives returning visitors something to notice. | Done |
| 4 | **Share button on each card** — Using the Web Share API (with clipboard fallback). Players frequently share court info with friends via Messenger/group chats — currently requires screenshotting. | To-Do |
| 5 | **Sort by popularity** — Leverage custom analytics data (e.g., total click events per listing) to rank courts by engagement. Requires aggregating analytics entries per listing. | To-Do |
| 6 | **Sort by city** — Alphabetical city-based sorting as an alternative to the existing city filter dropdown. | To-Do |
